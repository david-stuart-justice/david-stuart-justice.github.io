Event Sourcing

Event Sourcing: Concepts, Architecture, and Implementation

Event sourcing stores state as an append-only sequence of events. This collection covers conceptual foundations, architecture patterns, implementation approaches (Axon Framework vs manual), and decision frameworks.

What is Event Sourcing?

A Persistence Pattern

Event sourcing is a persistence approach where state is not stored as a current snapshot, but as a complete sequence of events that led to the current state. To get the current state of an aggregate, you replay all its events from the beginning.

Traditional CRUD

  • Store current state in database rows
  • Updates overwrite previous values
  • History is lost unless explicitly tracked
  • Simpler for most use cases

Event Sourcing

  • Store every state change as an event
  • Events are immutable, append-only
  • Complete audit trail by design
  • Adds complexity, use when benefits justify cost

Framework Approach

Axon Framework

A complete event sourcing and CQRS framework that provides aggregate loading, concurrency control, and tracking processor management.

  • Automatic state reconstruction from events
  • Built-in concurrency and conflict detection
  • Structural outbox via event store tracking
  • Advanced testing with AggregateTestFixture

Manual Event Sourcing
